# AI-Powered Daily Business Report Generator

Turn your daily sales data into a polished, AI-written executive report — delivered straight to your inbox.

A Python tool that reads e-commerce sales data, computes key business metrics, generates a natural-language summary using the Gemini AI API, and emails the report to a list of recipients. Designed to be scheduled (Cron, Task Scheduler) for automated daily delivery.

## The Problem

Most small and mid-sized businesses drown in dashboards. They have data, but turning that data into *insights* requires someone to look at numbers, spot trends, write up findings, and send them out — every single day.

This tool automates that loop. Raw data in, executive-ready report out, no human intervention required.

---

## How It Works

sales_data.csv
↓
[1] metrics.py        → Calculates daily, weekly, and top-performer metrics
↓
[2] ai_summarizer.py  → Sends structured metrics to Gemini for natural-language summary
↓
[3] email_sender.py   → Delivers the report via Gmail SMTP
↓
Your inbox at 9 AM

---

Each module is independent and reusable. You can swap the data source (CSV → MySQL, Postgres, BigQuery), the AI provider (Gemini → Claude, GPT, etc.), or the delivery channel (email → Slack, Discord, webhook) without touching the others.

## Setup

### 1. Clone and install

```bash
git clone git@github.com:amangupta-py/ai-business-report-generator.git
cd ai-business-report-generator
python -m venv venv
source venv/bin/activate          # On Windows: .\venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### 2. Configure credentials

Copy `.env.example` to `.env` and fill in your values:

```bash
cp .env.example .env
```

You'll need:

- **Gemini API Key** — Free from [aistudio.google.com](https://aistudio.google.com)
- **Gmail App Password** — Generate at [myaccount.google.com/apppasswords](https://myaccount.google.com/apppasswords) (requires 2-Step Verification enabled)
- **Recipient email(s)** — Comma-separated list

### 3. Generate sample data (optional)

If you want to test with sample data:

```bash
python generate_sample_data.py
```

This creates `sales_data.csv` with 30 days of realistic e-commerce data.

## Usage

**Run the full pipeline (loads data, generates summary, sends email):**

```bash
python main.py
```

**Test mode (prints to terminal, doesn't send email):**

```bash
python main.py --no-email
```

**If needed schedule it daily (Linux/Mac, runs at 9 AM):**

```bash
0 9 * * * cd /path/to/project && /path/to/venv/bin/python main.py >> log.txt 2>&1
```

## Design Decisions

A few intentional choices worth noting:

- **Modular structure** — Each file does one job. Swap any layer (data, AI, delivery) without touching the others.
- **Defensive error handling** — API failures, missing config, and empty responses raise clear errors, not silent failures. Important when scheduled.
- **Secrets via `.env`** — No credentials committed to git. `.env.example` documents the structure for new users.
- **JSON-formatted prompts** — Metrics passed to the AI as structured JSON, not prose. Produces tighter, more accurate output.
- **Configurable recipients** — Recipients live in `.env`, not hardcoded. Easy to deploy across teams without code changes.
